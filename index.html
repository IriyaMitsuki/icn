Начало работы с Git и GitHub

Это руководство поможет вам сделать первые шаги в мире контроля версий. Мы разберем основные понятия и команды, которые необходимы каждому разработчику.

Git — это распределённая система управления версиями. Проще говоря, это инструмент, который отслеживает изменения в файлах вашего проекта с течением времени. Он позволяет вам возвращаться к предыдущим версиям, сравнивать изменения и работать над проектом совместно с другими людьми.

GitHub — это веб-сервис для хостинга Git-репозиториев. Он предоставляет облачное хранилище для ваших проектов, а также инструменты для совместной работы, отслеживания задач, проведения код-ревью и многого другого. GitHub — это социальная сеть для разработчиков.

Шаг 1: Установка и настройка

Прежде всего, вам нужно установить Git на ваш компьютер. Это можно сделать, скачав установщик с официального сайта Git. Процесс установки прост и не требует особых настроек.

После установки откройте терминал (командную строку) и выполните следующие команды, чтобы представиться системе Git. Это важно, так как эта информация будет прикреплена ко всем вашим будущим коммитам.

# Указываем ваше имя
git config --global user.name "Ваше Имя"

# Указываем ваш email, который вы использовали при регистрации на GitHub
git config --global user.email "ваш@email.com"

Также не забудьте создать аккаунт на GitHub.com, если у вас его еще нет.

Шаг 2: Создание первого репозитория

Репозиторий (или "репо") — это, по сути, папка вашего проекта со всеми файлами и историей их изменений. Давайте создадим свой первый репозиторий на GitHub.

  1. Войдите в свой аккаунт на GitHub.
  2. В правом верхнем углу нажмите на значок `+` и выберите New repository.
  3. Дайте вашему репозиторию имя (например, `hello-world`), добавьте краткое описание.
  4. Вы можете сделать репозиторий публичным (виден всем) или приватным (видны только вам и тем, кому вы дадите доступ).
  5. Рекомендуется поставить галочку напротив Add a README file. Это создаст начальный файл с описанием проекта.
  6. Нажмите кнопку Create repository.

Поздравляем! Ваш первый репозиторий создан на GitHub.

Шаг 3: Основной рабочий процесс

Теперь давайте скопируем репозиторий с GitHub на ваш компьютер, внесем изменения и отправим их обратно.

Клонирование репозитория

Чтобы работать с файлами локально, нужно "клонировать" удаленный репозиторий. На странице вашего репозитория на GitHub нажмите зеленую кнопку <> Code, скопируйте URL (убедитесь, что выбрана вкладка HTTPS) и выполните в терминале команду:

# Замените URL на URL вашего репозитория
git clone https://github.com/ВАШ_НИКНЕЙМ/ИМЯ_РЕПОЗИТОРИЯ.git

# Перейдите в папку созданного проекта
cd ИМЯ_РЕПОЗИТОРИЯ

Коммит и Push

После того как вы внесли изменения в файлы проекта, их нужно сохранить в Git и отправить на GitHub. Этот процесс состоит из трёх шагов:

1. Добавление в индекс (Staging): Вы сообщаете Git, какие именно изменения хотите включить в следующий "снимок" (коммит).

# Добавляем все измененные и новые файлы в "индекс"
# Точка означает "все файлы в текущей директории"
git add .

2. Фиксация (Commit): Вы создаете тот самый "снимок" с осмысленным сообщением, описывающим сделанные изменения.

# Фиксируем изменения с осмысленным сообщением
git commit -m "Добавил информацию о рабочем процессе в README"

3. Отправка на GitHub (Push): Ваш коммит пока существует только на вашем локальном компьютере. Чтобы он появился на GitHub, выполните команду `push`:

# Отправляем ваши коммиты в удаленный репозиторий (origin) в ветку main
git push origin main

Теперь, если вы обновите страницу вашего репозитория на GitHub, вы увидите все внесенные изменения!

Следующие шаги: Ветвление (Branching)

Ветвление — это одна из самых мощных функций Git. Она позволяет создавать изолированные "ветки" для разработки нового функционала или исправления ошибок, не затрагивая основную, стабильную версию кода (которая обычно находится в ветке `main`).

Это похоже на создание копии проекта, в которой вы можете безопасно экспериментировать.

# Создать новую ветку с именем new-feature
git branch new-feature

# Переключиться на новую ветку, чтобы начать в ней работать
git checkout new-feature

# Или создать и сразу переключиться на нее одной командой
git checkout -b another-new-feature

Pull Requests

Pull Request (PR или запрос на слияние) — это способ предложить ваши изменения для включения в другую ветку (чаще всего в `main`). Это центральный элемент совместной работы на GitHub.

Когда вы создаете Pull Request, вы даете возможность другим участникам команды просмотреть ваш код (провести код-ревью), оставить комментарии и убедиться, что ваши изменения не сломают проект, прежде чем они будут влиты в основную кодовую базу.

Совместная работа

GitHub делает совместную работу над кодом простой и эффективной. Используя ветки и pull requests, команда может одновременно работать над разными задачами без конфликтов. Каждый разработчик работает в своей ветке, а затем предлагает свои изменения через Pull Request. Это обеспечивает прозрачность, контроль качества и позволяет создавать сложные проекты силами большой команды.
